Introduction

New Years Resolutions are made by people every year and while some stick to them, others do not. We thought it would be interesting to visualize the types of resolutions that were made in 2015 heading into 2016. The aim of this project is for us to visualise data with about 3 to 4 dimensions. We wanted to enable users have an overview of all the data points in the data set and identify patterns. Our second aim was for the user to be able to interact with the data to get answers to their visual queries. They should be able to drill down to lower levels of dimensionality and interact with the discreet data. To achieve these aims we used an interactive visualisation software called D3js, which is a javascript visualisation library.

Our Data

We got the data off a CrowdFlower blog about new year resolutions. The data was scrapped off of Twitter, taking anything that was tweeted from the United States with a 'hashtag' of #NewYearsResolution. The data that they collected was compiled into a csv file with the following columns: Twitter handle, gender, tweet, resolution category, resolution topic (which narrows down the category), 'retweet' count, tweet time and date, location coordinates and state.

Research

Before the initial design was developed, research into existing visualizations on the same topic was done. All the visualizations discovered were static and the data was not multi-dimensional. Below are some of the visualizations along with an analysis for each of them.

Example 1: New Jersey New Year's Resolution

The visualization associated a different colour with each category in the pie chart. This made it easy to get the know which categories were most popular. Though pie charts are considered bad visualizations(especially for multiple categories), this one shows the value(percentage) for category each clearly. This helps especially when looking at the poorer performing categories. Without this extra information, it would difficult to tell which slice of the pie chart is smaller than the other.

There is no discrete information displayed. Therefore so one does not know how big the data set they was used is or if the data was a true representation of the new year's resolutions of the people in New Jersey. Lastly, one cannot drill down on the data to see specific numbers about the categories.

Visual Queries

With our visualization, we want the user to be able to answer the following questions:

  • What is the distribution of tweets per category throughout the nation or in a state ?
  • How do the genders compare in different categories ?
  • How do any 2 given states compare overall or in a particular category? ?
  • What is the most popular category in a state?
  • What is the spread of tweets per gender throughout the nation or in a given state ?

Design Iterations

Initial Design

Figure 6 : The initial design as presented to the class

The initial design included a map and supporting graphs (below) to show discrete information about the data. The map shows all the data points in the data set at the same time and how they are the spread across the nation and in each state. Each point on the map represents a tweet and the location from which it was tweeted. Users can get all the information about a data point by hovering over it (shown on image above). The slider below the map allows users to select the range of time starting from the 21st of December 2015 to the 1st of January 2016.

Above the map, there are checkboxes with each one representing a category or a gender (male and female). This is to allow a user to select any number of categories or a specific gender they wish to view. The changes made in the selection are propagated to the map and graphs in order for them to show only the selected categories or gender.

The design caters for comparison between a maximum of two states, which can be done by selecting the two states from the two combo boxes above the map. Selecting one state would highlight the state (and grey out everything else) on the map and change the graphs to represent only that state. Selecting another state would then highlight both the states and display both the graphs of the states to allow for comparison between the two.

At this point, the choice of colours was made with a white background in mind. All the colours of the unique hue were selected with the exception of white and yellow. Because a white background was being used, these 2 colours could not be chosen as they would not be clearly visible. Therefore the colour yellow was replaced with orange and white with grey.

Second Iteration

The second iteration was developed using D3js. The visualization is interactive and contains all the data points.

Figure 9:


Figure 10:

Implementation of feedback (from 1st presentation) in 2nd iteration

Use shape as well as colour to represent a category

We thought the colors representing the categories were so distinct that a person did not need an additional symbol to help them easily identify a category. Due to the fact that we have a large data set we decided not to use symbols for categories because we felt they would only overcrowd the map even more. We decided use a circle to represent data points and have them colour coded to the category it falls under. An extra dimension of gender was added by having male data points have a black border around the circle, while female data points have a white border.

Comparing two states

Users have the ability to compare only two states at a time. A state is chosen by selecting it from the combo box above the map. This changes the map as well as the graphs, therefore providing discrete information. Comparisons can be made within a single state across its different categories or genders. Comparisons can also be made between two states across all of their categories or in one specific category.

Use both texture and colour for bar graphs

The graphs use both colour and texture. Each category retains its colour while texture is added to the bar representing females as clearly shown on the key.

Have a bit more of theme - resolutions - add appeal to your graphic

Rather than adding the new year's theme to our visualisation through symbols and colour etc., we decided to add the theme to the web page instead. We wanted a visually appealing visualisation that did not overwhelm the user with unnecessary symbols and colour. Our aim is for the user to easily understand the visualization and have their visual queries answered as fast as possible.

Total resolution (category) "winner" of the state shown as a texture?

At this stage had not added this feature because one category was the winner in about 47 of the 50 states. We thought this would just fill up the entire map with the symbol or color and decided to think about it for final implementation. A smaller map would be added next to the large map showing this. This map would look like the one showed below. We decided to make a decision on this during the final implementation.

Figure 11: A map showing the 'winning' category in each state

Symbols were discarded for this iteration because they were only going to be used on the key and therefore served no major purpose. The slider showing time ranges was discarded because of the short time frame it covered. This iteration is interactive and users get more information by interacting with the visualization in different ways:

  • Hovering over a data point on the map shows all the information about that point.
  • Clicking on a state in the combo box. This removes all the data points on the rest of map and only leaves the ones in that specific state. The graphs change accordingly. This could be improved by greying out the other points instead of removing them completely so that a user still has an overview.
  • Clicking on a second state would show the data points in this state, change the graphs and therefore enable users to make comparisons between the two states. This iteration caters for comparisons between only 2 states.
  • Clicking on a gender selects or un-selects the gender depending on its current state(selected or un-selected). This adds or removes the data points (for that gender) on the map and changes the graphs as well. Selecting categories works exactly like selecting the gender.

  • Use a 2 different shapes - one for males and one for females

  • Add more appeal (theme) to the visualization

Discussion

The final implementation is very similar to our initial design. This was expected as we did not get a lot of negative feedback after our first presentation. With this visualization, we are able to:

  • Show all the information about each tweet and its location on the map
  • View all the information for only one state or category or gender
  • Compare any two states
  • Compare any number of categories at the same time
  • Compare the two genders by each category and or state

Strengths

The visualization has an overview (map) of all the data in the data set and has graphs which show discrete information. The graphs make it easy to comparisons from the visualization. Furthermore, the visualization covered the 3 main categories discussed by Ben Shneiderman in his visualization mantra. These are Overview, Zoom and Filter and lastly, Details on Demand. The map provided the overview while the interactivity of the visualisation provided the other 2.

Overview:

The map overview (before any interaction) allows a user to see all the data points at the same time. From this, the user can notice patterns or see which categories are popular in which areas.

Zoom and Filter:

Users can zoom into a particular state to see the information for only the selected state. The graphs change to show discrete information, such as number of resolutions in each category, for that particular state. Using the same functionality, users can make comparisons between two states.

Filtering can be done on both the categories and the genders. This allows users to see information about a specific category in a specific state or on the entire map. Filtering also changes the graphs to enable users get more information about the category or gender they have selected.

Details on demand:

Clicking on a point on the map shows all the information about that particular data point on the map. This allows a user to get all the information about that tweet while having an overview of the entire data set in the background.

Weaknesses

The major weakness of our visualization is the poor spacing. The East of the map has significantly more data points. There is nothing we could about this.

Secondly, most of the queries require the user to interact with the visualization.

The states are not labelled which makes it hard for people who are not familiar with the map of USA to know the location of each state or which state they are looking at without selecting one.

There are no "all" and "nothing" buttons or checkboxes. The "all" button would enable a user select all the categories at once without having to check all the 9 checkboxes. And similarly, the "nothing" button would enable a de-select all the catgories and choose the one they want to explore.
